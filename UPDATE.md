# 更新日志

## v2.2.4 (2026-06-11)

### 支付二维码弹窗修复

- **修复二维码弹窗被控制面板遮挡**：弹窗 z-index 从 999999 提升至 2000000030，确保始终显示在控制面板之上（根本原因）

### 详情弹窗增强

- **锁单成功时显示「重新弹出支付二维码」按钮**：点击接口记录查看详情，底部可直接重新弹出支付二维码
- **QRCode 库加载失败时的 fallback**：引导用户复制 sign 链接，并跳转 [在线二维码生成工具](https://freetoolkit.cn/tools/%E4%BA%8C%E7%BB%B4%E7%A0%81%E7%94%9F%E6%88%90) 手动生成
- **Preview 成功未锁单时显示「打开支付页面」按钮**：触发页面原生支付弹窗
- **详情弹窗支持文本选择**：修复拖拽逻辑的 `preventDefault` 阻止文本选择的问题，现在可以正常复制内容

### QR 弹窗简化

- 移除冗余的「重新生成」按钮，URL 区域点击可复制

## v2.2.3 (2026-06-10)

### 预存优化

- **默认预存数量调整为 1**：降低默认预存量，减少验证码频率限制风险
- **最大预存数量限制为 5**：防止过多预存触发反作弊机制
- **自动预存时机提前至 30 秒**：倒计时 ≤ 30s 时自动触发预存，确保 token 在有效期内被使用
- **LIFO（后进先出）消耗策略**：预存 token 按后进先出顺序使用，最新获取的 ticket 优先消耗，有效期更长
